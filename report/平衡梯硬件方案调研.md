## 1. 项目定位

平衡梯是一个用于自动控制原理课程设计的主动平衡对象。它不是实际可用的梯子，而是一个结构上故意不稳定的控制实验平台：H 型 PCB 竖直放置，底部用尖脚形成近似线接触支撑，中上部的电机和螺旋桨产生可控推力，控制器根据姿态反馈实时调节电机，使系统维持在目标角度附近。

该项目能对应课程中的几个核心概念：

| 自动控制概念 | 在本项目中的体现 |
|---|---|
| 开环不稳定 | 不加控制时，H 型 PCB 会自然倒下 |
| 闭环反馈 | MPU6050 测得角度，ESP32-C3 根据误差调节电机 |
| 执行器饱和 | 电机推力、反转响应和 PWM 上限限制控制效果 |
| PID 调参 | `Kp/Ki/Kd` 改变响应速度、超调和稳态误差 |
| 抗扰动 | 手推、桌面振动、重心变化都会反映到闭环响应 |
| 目标跟踪 | 目标角可从 90 deg 改到 85 deg，观察系统跟随能力 |

## 2. 最终硬件架构

```text
18650 电池
  └── FM5324H 充放电/升压
        ├── +5V ── ESP32-C3 SuperMini 5V
        ├── +5V ── DRV8837 VM ── 8520 电机
        └── 指示灯/按键/Type-C 充电口

ESP32-C3 SuperMini
  ├── MPU6050：姿态测量
  ├── OLED：状态显示
  ├── DRV8837：电机正反转 PWM 控制
  ├── GPIO1 按键：低电平有效，扶正后解锁闭环
  ├── BAT_ADC：电池电压检测
  ├── BLE：姿态遥测和调参指令
  └── UART0：外部调试接口预留
```

最终原理图按功能分成 `Power`、`MCU`、`Display&Sensor`、`Motor` 四块。PCB 上 H 型结构本身就是承力主体，电池在下方降低重心，电机在中上部增加推力力臂，ESP32-C3、DRV8837、FM5324H 和 OLED 集中在下半部分便于布线和调试。

## 3. 最终器件选型

| 模块 | 最终选型 | 说明 |
|---|---|---|
| 主控 | ESP32-C3 SuperMini | 小尺寸、带 USB 下载和 BLE，算力足够运行 200 Hz 以上闭环 |
| 姿态传感器 | MPU6050 模块 | 三轴加速度计 + 三轴陀螺仪，I2C 通信，适合课程设计 |
| 显示 | 0.96 inch I2C OLED | 用于显示角度、电池电压、PWM、模式和故障状态 |
| 电机 | 8520 空心杯电机 | 推力余量比 820 更充足，适合 H 型 PCB 结构 |
| 电机驱动 | DRV8837 | 单路 H 桥，正好匹配单电机正反转方案 |
| 电池 | 单节 18650 锂电池座 `BT1` | 原理图封装为 `BH-18650-B1BA002` |
| 电源芯片 | FM5324H | Type-C 输入，单节锂电充电，升压输出 +5 V |
| 充电口 | USB Type-C `USB1` | `CC1/CC2` 各接 5.1 kΩ 下拉，作为 5 V Sink |
| 调试接口 | `H1` 1x4 排针 | 预留 `VCC/GND/TX/RX` |
| 底部尖脚 | `H2/H3` 1x3 排针 | 用作底部尖脚/支撑接触点 |

## 4. ESP32-C3 引脚分配

最终原理图中的网络和 ESP32-C3 SuperMini 引脚对应如下：

| 功能 | ESP32-C3 引脚 | 原理图网络 | 连接目标 |
|---|---:|---|---|
| 电池电压 ADC | GPIO0 | `BAT_ADC` | R3/R5 分压后的电池电压 |
| 启动/解锁按键 | GPIO1 | 按键输入 | 低电平有效，扶正后按下才允许闭环工作 |
| MPU6050 中断 | GPIO2 | `MPU_INT` | 经 `R7=10 kΩ` 连接到 MPU6050 `INT` |
| MPU6050 SCL | GPIO3 | `MPU_SCL` | MPU6050 `SCL` |
| MPU6050 SDA | GPIO4 | `MPU_SDA` | MPU6050 `SDA` |
| DRV8837 休眠控制 | GPIO5 | `DRV8837_nSLEEP` | DRV8837 `nSLEEP` |
| 电机输入 1 / PWM | GPIO6 | `DRV8837_IN1` | DRV8837 `IN1` |
| 电机输入 2 / PWM | GPIO7 | `DRV8837_IN2` | DRV8837 `IN2` |
| OLED SDA | GPIO9 | `OLED_SDA` | OLED `SDA` |
| OLED SCL | GPIO10 | `OLED_SCL` | OLED `SCL` |
| 外部串口 RX | GPIO20 | `UART0_RX` | H1 调试排针 `RX`，备用调试 |
| 外部串口 TX | GPIO21 | `UART0_TX` | H1 调试排针 `TX`，备用调试 |

需要注意：最终原理图把 MPU6050 和 OLED 分成了两组不同的 I2C 网络。ESP32-C3 通常只有一组硬件 I2C 控制器，因此软件上建议 MPU6050 使用硬件 I2C，OLED 使用软件 I2C，或者后续硬件改版时把两者合并到同一组 I2C 总线上。

## 5. DRV8837 电机控制

DRV8837 没有单独的 `PWM` 引脚，PWM 直接加到 `IN1` 或 `IN2`。最终硬件使用 `GPIO6/GPIO7` 分别连接 `IN1/IN2`，`GPIO5` 控制 `nSLEEP`。

| 控制状态 | IN1 | IN2 | 说明 |
|---|---|---|---|
| 正向推力 | PWM | LOW | 电机正转 |
| 反向推力 | LOW | PWM | 电机反转 |
| 滑行停机 | LOW | LOW | 输出高阻，电机自然滑行 |
| 刹车停机 | HIGH | HIGH | 两端短接刹车 |

单电机正反转方案的风险是螺旋桨反向推力通常弱于正向推力，且从正转切到反转存在响应延迟。课程设计可以接受这个非理想性，因为它本身也体现了执行器饱和、死区和动态滞后对闭环系统的影响。软件上需要加入 PWM 限幅、死区补偿和方向切换保护。

## 6. 电源与电池检测

最终电源方案使用 `FM5324H` 对单节 18650 电池进行充电和升压。板上有独立的 Type-C 充电口 `USB1`，同时 ESP32-C3 SuperMini 模块自身也有 USB-C 口用于下载程序和日志查看。调参和姿态遥测主要通过 BLE 完成，H1 串口作为备用调试接口。

电源网络关系：

| 网络 | 来源 | 主要负载 |
|---|---|---|
| `BAT` | 18650 电池座 | FM5324H 输入、电池电压分压 |
| `+5V` | FM5324H 升压输出 | ESP32-C3 SuperMini `5V`、DRV8837 `VM` |
| `VCC` | ESP32-C3 SuperMini `3V3` | MPU6050、OLED、DRV8837 逻辑电源、调试排针 |
| `GND` | 系统地 | 全板公共地 |

电池 ADC 分压由 `R3=33 kΩ`、`R5=100 kΩ` 和 `C8=100 nF` 组成：

```text
BAT+ -- 33 kΩ -- GPIO0 ADC -- 100 kΩ -- GND
                    |
                  100 nF
                    |
                   GND
```

计算关系：

```text
Vadc = Vbat * 100 / (33 + 100) = Vbat * 0.752
Vbat = Vadc * (33 + 100) / 100 = Vadc * 1.33
```

单节锂电满电 `4.2 V` 时，ADC 输入约为：

```text
Vadc = 4.2 * 0.752 = 3.16 V
```

该值低于 ESP32-C3 的 3.3 V ADC 输入范围，并留有一定余量。分压静态电流约为 `4.2 / 133 kΩ = 31.6 uA`，对 18650 电池影响很小。

## 7. 姿态测量和安装注意

MPU6050 不要求“平放”才能用。最终项目中 MPU6050 模块跟随 H 型 PCB 竖直安装，软件只需要在初始化时确定传感器坐标轴和控制角度之间的对应关系。

安装和调试要点：

- MPU6050 必须固定在刚性较好的 PCB 区域，不能只粘在 OLED 或软连接结构上。
- OLED 和 MPU6050 可以上下叠放，但要保证绝缘、留出间隙，并避免 OLED 排针顶到 MPU 模块。
- MPU6050 和 OLED 均使用 `VCC=3.3 V` 逻辑，I2C 上拉不能拉到 5 V。
- 电机、电感、升压电路尽量远离 MPU6050，减少振动和电源噪声耦合。
- 软件里不要假设模块平放，要通过静止标定记录零点和正方向。

## 8. ESP32-C3 天线和布板注意

ESP32-C3 SuperMini 使用板载陶瓷天线。陶瓷天线虽然比 PCB 天线更集中，但仍然需要天线区域净空：

- 天线所在端尽量靠近 H 型 PCB 边缘。
- 天线下方和前方不要铺铜、走高速线或放金属件。
- 电池、OLED、电机和升压电感尽量远离天线。
- 如果 BLE 仅用于近距离调参，天线性能下降主要影响连接距离和稳定性；如果后续要稳定无线遥测，就需要更严格的天线净空。

## 9. 调试路线

建议按下面顺序调试，避免一上来就闭环带桨满功率：

1. 只上电，不装电机和螺旋桨，确认 `+5V`、`VCC`、`BAT_ADC` 正常。
2. 下载 ESP-IDF 最小固件，确认 ESP32-C3 能启动，日志能输出。
3. 读取 GPIO1，确认未按下为高电平、按下为低电平。
4. 扫描 MPU6050 和 OLED，确认两组 I2C 都可用。
5. 建立 BLE 连接，确认能上传姿态信息、能接收目标角和 PID 参数。
6. 静止读取 MPU6050，完成陀螺仪零偏和角度方向确认。
7. 不装螺旋桨测试 DRV8837，确认 `IN1/IN2/nSLEEP` 和电机正反转。
8. 装螺旋桨后限幅测试推力，记录正反转推力差异。
9. 手持平衡梯做闭环输出检查，确认倾斜方向和电机修正方向相反。
10. 扶正平衡梯，短按 GPIO1 后低 PWM 上限站立调参，先调 `Kp/Kd`，最后再考虑 `Ki`。
11. 加入角度超限停机、低电压停机、BLE 停机命令和按键急停。

## 10. 主要风险

| 风险 | 表现 | 规避方式 |
|---|---|---|
| 反向推力不足 | 一侧能拉回，另一侧拉不回 | 使用对称桨/更大桨，软件做方向补偿 |
| 电机噪声影响 IMU | 角度跳变、I2C 异常 | 电源去耦、线短、地回路分区、滤波 |
| GPIO2/GPIO9 启动影响 | 上电或下载异常 | `GPIO2` 可改为轮询不用 INT；`GPIO9` 检查 OLED 上拉，必要时断开 OLED 再下载 |
| 按键误触发 | 未扶正就启动闭环 | GPIO1 只作为解锁条件之一，还必须要求角度接近目标且电池/IMU 状态正常 |
| PCB 结构振动 | PID 调不稳，高频抖动 | 降低控制带宽，增加结构刚性 |
| 底部尖脚打滑 | 平衡点横向移动 | 更换接触面，调整尖脚间距和高度 |
| 螺旋桨危险 | 调参时突然高速旋转 | 先无桨调试，闭环阶段限幅并设置急停 |
