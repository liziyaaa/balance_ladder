# 平衡梯硬件方案调研

> 更新时间：2026-06-07  
> 目标：用于自动控制原理课程设计，确定“PCB H 型平衡梯 + ESP32-C3 + 空心杯电机螺旋桨”的硬件路线。

## 1. 结论先行

推荐先按“模块验证，PCB 结构集成”的路线做：

1. **主控**：ESP32-C3 SuperMini 或 ESP32-C3-MINI-1 模块。课程设计优先用 SuperMini，降低射频、USB 下载和焊接风险；最终 PCB 可以做插座/邮票孔焊盘。
2. **姿态传感器**：确定使用 MPU6050。首版建议直接用 GY-521 模块，PCB 上预留 4P/5P 插座；如果后续想高度集成，再把 MPU6050 芯片本体画到板上。
3. **执行器**：1 个 820/8520 空心杯电机 + 螺旋桨。由于要左右两侧都能纠偏，电机驱动必须支持**正反转**，螺旋桨最好选可逆/对称桨，或至少准备正桨、反桨做实测。
4. **电机驱动**：确定使用 DRV8833。它是双 H 桥驱动，既可以只用 A 通道驱动 1 个空心杯电机，也可以把 A/B 两个通道并联来提高电流余量。
5. **供电**：1S LiPo 450-650 mAh 高倍率电池。调试版可用 1S + 5 V 升压给 C3 和电机驱动；最终版更推荐“电机直接吃电池电压 + 3.3 V 稳压给控制电路”，效率更高、噪声更可控。
6. **结构**：H 型主体用 PCB 直接制作可行，但建议用 2.0 mm 或 2.4 mm FR-4；底端尖点不要只靠 FR-4 边缘，最好做成可更换的金属尖脚。

最小可行硬件不是只有“ESP32-C3 + 电机 + MPU6050”。还需要：H 桥电机驱动、电源管理/充电保护、3.3 V 稳压或 5 V 升压、电池电压检测、USB-C 充电口、总电源开关、滤波电容、调试接口、尖脚结构件、螺旋桨防护圈。

## 2. 系统工作方式

平衡梯可以看成一个绕底部两尖点连线转动的倒立摆。H 型 PCB 的两个底端尖点形成一条支撑线，梯子主要在垂直于这条线的方向上倾倒。

```
侧视控制平面

        螺旋桨水平吹风
             ⇄ F
             |
          [电机]
             |
       ┌─────┴─────┐
       │           │
       │  PCB H型体 │   θ：偏离竖直角度
       │           │
       └───┬───┬───┘
           ▽   ▽
       两个尖点形成支撑线
```

推力在高处产生回复力矩：

```text
推力力矩 ≈ F * h_p
重力倾倒力矩 ≈ m * g * h_cm * sin(theta)
```

因此选电机时不要只看转速，要看“推力 × 安装高度”是否足够。建议按下面的估算留 2 倍以上余量：

```text
F_min = m * g * h_cm * sin(theta_max) / h_p
```

例子：总质量 180 g、质心高度 120 mm、螺旋桨高度 220 mm、希望 10° 内能拉回：

```text
F_min ≈ 0.18 * 9.81 * 0.12 * sin(10°) / 0.22
      ≈ 0.167 N ≈ 17 g 推力
```

考虑电机响应、风扇效率、反向推力效率和电池掉电，建议实测最大推力至少 **30-40 g**。如果 PCB 主体、电池和电机总重超过 150 g，8520 比 820 更稳妥；如果总重能压到 100-120 g，820 更轻、更容易做小。

## 3. 推荐硬件架构

### 3.1 原型验证版

适合先跑通控制闭环。

```text
1S LiPo
  ├─ 充电/保护模块
  ├─ 5V升压 ── ESP32-C3 SuperMini 5V
  └─ 5V升压或电池直供 ── H桥驱动 VM ── 空心杯电机 + 螺旋桨

ESP32-C3
  ├─ I2C ── GY-521 / MPU6050
  ├─ 双 PWM 输入 ── DRV8833 H桥驱动
  ├─ ADC ── 电池电压采样
  └─ USB/WiFi/BLE ── 下载、串口、无线调参
```

优点是调试快，缺点是升压模块和杜邦线会引入噪声、重量和结构松动。

### 3.2 最终 PCB 版

适合课程展示。

```text
USB-C充电口
  └─ 1S锂电充电管理 + 保护
        └─ LiPo电池
              ├─ 电机电源 VM：电池直供或固定5V升压
              └─ 控制电源：3.3V稳压/升降压 ── ESP32-C3 + MPU6050

ESP32-C3
  ├─ I2C ── MPU6050
  ├─ LEDC PWM ── DRV8833 AIN1/AIN2
  ├─ GPIO ── nSLEEP 使能 / nFAULT 故障输入
  └─ ADC ── 电池电压、电机电流可选
```

如果使用 ESP32-C3 SuperMini 成品板，最终 PCB 可以把它当“可插拔主控模块”。如果用 ESP32-C3-MINI-1 官方模组直接贴片，PCB 会更干净，但要按射频布板规则做天线净空。

## 4. 关键器件调研与选型

### 4.1 主控：ESP32-C3

ESP32-C3 是单核 32 位 RISC-V MCU，最高 160 MHz，带 2.4 GHz Wi-Fi 和 Bluetooth LE 5，官方数据手册列出 400 KB SRAM、I2C、USB Serial/JTAG、LEDC PWM 等外设。对平衡梯来说，它的算力足够跑 200-500 Hz 的姿态读取、互补滤波和 PID/LQR 控制。

推荐：

| 方案 | 适合阶段 | 优点 | 风险 |
|---|---|---|---|
| ESP32-C3 SuperMini | 原型和课程展示 | 便宜、小、Type-C 下载方便 | 不同商家板子质量和天线差异大 |
| ESP32-C3-MINI-1 模组 | 最终 PCB | 官方模组，射频更可靠 | 焊接和 PCB 设计要求更高 |
| 裸 ESP32-C3 芯片 | 不推荐首版 | 成本低、集成度最高 | 需要射频、晶振、Flash、电源完整设计 |

布板提醒：

- 用模组时，把 PCB 天线放在板边，天线区域避免铜皮、走线和器件。
- 如果天线不能伸出底板，天线四周至少留出约 15 mm 净空。
- 电机、电感、升压芯片、USB 口尽量远离天线和 MPU6050。
- ESP32-C3 的 LEDC 可产生 PWM，用于电机控制；实际 PWM 频率建议 18-25 kHz，避开可闻噪声。

按你截图里的 **ESP32-C3 Super Mini** 板子，需要额外注意：

- 这块板使用 ESP32C3FN4，常见参数是 400 KB SRAM、384 KB ROM、内置 4 MB Flash。
- 引脚图右侧把 `GPIO8/GPIO9` 标成 `SDA/SCL`，但这只是板商给的常用丝印，不代表只能用这两个脚做 I2C。ESP32-C3 的 I2C 可以通过 GPIO Matrix 映射到其他 GPIO。
- 截图明确写了板载蓝色 LED 在 `GPIO8`，所以本项目不建议把 `GPIO8` 用作 MPU6050 的 SDA。
- `GPIO2/GPIO8/GPIO9` 是 ESP32-C3 的启动配置相关脚，外接电路不要在上电复位时强拉高或强拉低。本项目可以用 `GPIO2` 接 MPU6050 `INT`，但建议串 1 kΩ 或预留 0 Ω 电阻，方便调试时断开。

### 4.2 姿态传感器：MPU6050

姿态传感器确定使用 **MPU6050**。它集成三轴陀螺仪和三轴加速度计，I2C 接口，资料多、模块便宜，足够完成课程设计中的角度测量、互补滤波和 PID 控制。

推荐使用方式：

| 项目 | 建议 |
|---|---|
| 模块 | GY-521 MPU6050 模块 |
| 通信接口 | I2C |
| I2C 地址 | `AD0` 接 GND 时为 `0x68` |
| 供电 | 优先接 3.3 V，避免 I2C 上拉到 5 V |
| SDA | ESP32-C3 `GPIO4` |
| SCL | ESP32-C3 `GPIO3` |
| INT | 可选接 ESP32-C3 `GPIO2` |

安装要点：

- MPU6050 放在 PCB 中心附近、结构刚性好的位置。
- 不要贴近电机、螺旋桨和升压电感。
- I2C 线短一些，SDA/SCL 上拉到 3.3 V，建议 2.2 kΩ-4.7 kΩ。
- 预留 `INT` 脚，后面可以用数据就绪中断提高采样稳定性；不用中断时可不接。
- 控制轴要和 MPU6050 坐标轴明确对应，PCB 丝印上标出 X/Y/Z 和“前后倾倒方向”。
- GY-521 模块常带上拉电阻和稳压器；接线前确认模块原理图，首版优先让 I2C 全部工作在 3.3 V 电平。

### 4.3 执行器：空心杯电机 + 螺旋桨

这是本项目最容易低估的部分。单电机方案要解决两个问题：

1. **必须能产生双向推力**：梯子向前倒和向后倒时，需要相反方向的力矩。
2. **反向响应要快**：空心杯电机从正转刹停再反转有惯性和死区，PID 不能把它当理想力源。

建议选型：

| 总重目标 | 电机建议 | 桨叶建议 | 说明 |
|---|---|---|---|
| < 120 g | 820 空心杯 | 55-65 mm | 轻量优先 |
| 120-220 g | 8520 空心杯 | 65-75 mm | 成功率更高 |
| > 220 g | 不建议单 820 | 75 mm 以上或改双电机 | 单电机余量不足 |

硬件建议：

- 买正桨、反桨、对称桨各几片，实测正反转推力。
- 电机座要刚性固定，推力轴线必须穿过控制平面，避免产生偏航/侧向扭矩。
- 螺旋桨加防护圈。调 PID 时电机可能突然满占空比反转，裸桨不适合手边调试。
- 电机两端可并 0.1 µF 陶瓷电容抑制电刷噪声，驱动 VM 旁边放 100 µF 以上低 ESR 电容。

如果单电机反向推力效率太差，备选方案是“两个反向安装的小空心杯电机，一边负责向前吹，一边负责向后吹”。这会牺牲“一个电机”的设定，但闭环效果会明显更容易做出来。

### 4.4 电机驱动

空心杯电机是有刷 DC 电机，平衡梯需要 H 桥驱动来正反转。

| 驱动 | 电源范围/能力 | 适合性 | 备注 |
|---|---|---|---|
| DRV8833 | 2.7-10.8 V，约 1.5 A RMS/桥 | 确定使用 | 双 H 桥，可并联两路提高电流余量，有 nFAULT |
| L9110S | 小电流 | 不推荐首版 | 电流余量偏小 |
| L298N | 不推荐 | 不推荐 | 体积大、压降大发热，对小电池很不友好 |

DRV8833 没有单独的 `PWM` 引脚，PWM 直接打到输入脚。单电机使用 A 通道时：

```text
ESP32-C3 GPIO6  ── DRV8833 AIN1
ESP32-C3 GPIO7  ── DRV8833 AIN2
ESP32-C3 GPIO5  ── DRV8833 nSLEEP

DRV8833 AOUT1   ── 空心杯电机一端
DRV8833 AOUT2   ── 空心杯电机另一端
```

控制逻辑：

```text
正向推力：AIN1 = PWM，AIN2 = LOW
反向推力：AIN1 = LOW，AIN2 = PWM
滑行停止：AIN1 = LOW，AIN2 = LOW
刹车停止：AIN1 = HIGH，AIN2 = HIGH
```

PCB 注意：

- 电机电流回路要短而粗，和 MPU6050/I2C 走线分开。
- H 桥 VM 旁放 100 µF + 1 µF + 0.1 µF。
- 预留驱动故障脚到 ESP32-C3，便于过流/过温保护。
- 如果使用 DRV8833 并联输出，两路输入和输出要按数据手册建议并接：`AIN1` 与 `BIN1` 同信号，`AIN2` 与 `BIN2` 同信号，`AOUT1` 与 `BOUT1` 并接，`AOUT2` 与 `BOUT2` 并接。

### 4.5 电源与充电

推荐用 1S LiPo，因为重量比 18650 小很多。电机瞬时电流可能到 1-2 A，电池倍率不能太低。

| 模块 | 推荐 |
|---|---|
| 电池 | 1S 3.7 V LiPo，450-650 mAh，25C/30C，JST 插头 |
| 充电 | MCP73831、TP4056、IP5306 或成品 1S 充放电模块 |
| 保护 | 过充、过放、过流保护板，或选自带保护电池 |
| 控制电源 | 3.3 V 稳压，或 1S -> 5 V 升压后进 C3 开发板 |
| 电机电源 | 电池直供，或固定 5 V 升压后 PWM 限幅 |

两种实际可落地方案：

**方案 A：简单调试版**

```text
1S LiPo -> 充放电/升压模块 -> 5V
5V -> ESP32-C3 SuperMini 5V
5V -> H桥 VM
ESP32-C3 3.3V -> IMU + H桥逻辑电源
```

优点：接线简单，适合第一版。  
缺点：升压模块要承受电机脉冲电流，噪声较大；3.7 V 电机用 5 V 时要限制 PWM。

**方案 B：最终 PCB 版**

```text
1S LiPo -> H桥 VM -> 电机
1S LiPo -> 3.3V 升降压/LDO -> ESP32-C3 + IMU
USB-C -> 1S 锂电充电芯片 -> LiPo
```

优点：效率高，电机脉冲不经过 5 V 控制电源。  
缺点：如果用 SuperMini 的 5V 脚供电不方便，需要从 3.3V 脚供电并确认开发板电源路径。

USB-C 只做 5 V 充电输入时，CC1 和 CC2 都要各接一个 5.1 kΩ 到 GND，不能只接 VBUS/GND。

### 4.6 电池电压与安全

必须加的保护：

- 总电源滑动开关或拨码开关。
- 电池电压采样：电阻分压到 ESP32-C3 ADC，软件低电压停机。
- 电机急停逻辑：角度超过 35-45° 时立即停 PWM，防止摔倒后继续满功率吹。
- 螺旋桨防护圈。
- 充电口附近加 ESD/TVS 或至少预留焊盘。

可选增强：

- 电机电流采样：低边 0.05-0.1 Ω 采样电阻 + 运放/ADC，或 INA180/INA219。
- 蜂鸣器：低电压、摔倒、校准完成提示。
- OLED：调 PID 时显示角度、电池、PWM，但会增加重量。

## 5. PCB H 型结构建议

### 5.1 外形与材料

建议尺寸作为第一版参考：

| 项目 | 建议值 |
|---|---|
| 总高 | 220-300 mm |
| H 型宽度 | 120-180 mm |
| PCB 厚度 | 2.0 mm 或 2.4 mm |
| 材料 | FR-4，双面板即可；若集成射频/电源更完整可用四层 |
| 尖脚间距 | 与 H 型宽度一致，两尖点高度必须一致 |

PCB 做结构件时，最怕的是“看起来很酷，但太软”。H 型两条竖梁建议宽一些，中间横梁放电子元件，电机固定区加大铜皮和螺丝孔。必要时可以在背面加碳纤条、亚克力条或第二片 PCB 作加强。

### 5.2 尖脚方案

不建议只把 PCB 边缘画成尖三角，因为 FR-4 尖端容易磨损、分层，也难保证两个点等高。

更推荐：

| 方案 | 做法 | 优点 |
|---|---|---|
| 金属针脚 | PCB 底部焊接钢针/铜针 | 尖、耐磨、可调 |
| M2 尖头螺丝 | 底部做 M2 孔，装尖头螺丝 | 可更换、可调平 |
| 黄铜圆锥脚 | 焊盘或螺丝固定 | 外观好，展示感强 |
| PCB 三角脚 | 只作为外形，不承担最终接触 | 简单但寿命短 |

调试时建议在桌面放一块硬橡胶垫或亚克力板。尖点太滑会导致接触点平移，模型从“倒立摆”变成“滑动倒立摆”，PID 会很难调。

### 5.3 元件布局

推荐布局：

```text
顶部/中上部：电机 + 螺旋桨 + 防护圈
中部横梁：ESP32-C3、MPU6050、电机驱动、按键、LED
中下部：电池，可上下移动用于调重心
底部：两个金属尖脚，尽量对称
```

布局原则：

- 螺旋桨尽量高，增大力臂 `h_p`。
- 电池尽量低一些，降低质心高度 `h_cm`，减小所需推力。
- MPU6050 放在刚性区域，不要放在细长悬臂上。
- 电机和电池左右居中，避免重心偏向一边。
- 给电池做长圆孔或绑带槽，方便调重心。

### 5.4 布线原则

- 电机电源线、H 桥、升压电感走“大电流区”。
- ESP32-C3、MPU6050、I2C 走“信号区”。
- 大电流地和信号地最终单点汇接，不要让电机电流流过 MPU6050 地回路。
- I2C 不要贴着电机线走；必要时串 22-100 Ω 小电阻抑制尖峰。
- 所有模块旁边放测试点：3V3、5V/VBAT、GND、SDA、SCL、PWM_A、PWM_B、IN1、IN2、BOOT、EN。

| 调试 | 排针、测试点、LED、按键 | 若干 | 降低调试成本 |

## 7. 推荐引脚分配

下面按你截图里的 ESP32-C3 SuperMini 引脚图分配。原则是按核心板左右位置成组走线：左侧 `GPIO4/GPIO3/GPIO2` 给 MPU6050，右侧 `GPIO5/GPIO6/GPIO7` 给 DRV8833，底部 `GPIO20/GPIO21` 留给后续串口。

| 功能 | 建议 GPIO | 说明 |
|---|---|---|
| MPU6050 SDA | GPIO4 | 左侧相邻引脚；Arduino 里用 `Wire.begin(4, 3)` |
| MPU6050 SCL | GPIO3 | 左侧相邻引脚；可复用为普通 GPIO/I2C |
| MPU6050 INT | GPIO2 | 左侧相邻引脚；建议串 1 kΩ 或预留 0 Ω 电阻，方便断开 |
| DRV8833 nSLEEP | GPIO5 | 右侧相邻引脚；普通高低电平，高电平使能 |
| 电机输入 1 / PWM_A | GPIO6 | LEDC PWM 输出，接 DRV8833 `AIN1` |
| 电机输入 2 / PWM_B | GPIO7 | LEDC PWM 输出，接 DRV8833 `AIN2` |
| 电池 ADC | GPIO0 | 分压后输入；截图标 A0，适合测 1S 电池 |
| 串口 RX/TX 预留 | GPIO20/GPIO21 | 截图标 RX/TX，后续可接外部串口 |
| GPIO10 | 保留 | 不接 DRV8833 故障输入，留作备用 GPIO |
| 板载蓝灯 | GPIO8 | 不接外设，软件可作状态灯 |
| 保留/谨慎使用 | GPIO8/GPIO9 | 启动配置相关脚，首版不接关键外设 |
| BOOT/下载 | 保留板载按键 | 不要被电机驱动占用 |

`GPIO2` 是 ESP32-C3 的启动配置相关脚，但这里接的是 MPU6050 `INT`，不是 I2C 上拉或电机驱动输入，风险可控。硬件上建议让 `GPIO2 <-> MPU6050 INT` 经过 0 Ω 电阻位或 1 kΩ 串联电阻；如果后续发现上电/下载异常，可以直接不焊这颗电阻，软件改用轮询读取 MPU6050。

如果你后面一定想按截图丝印使用 `GPIO8/GPIO9` 做 I2C，也可以工作，但要确认 GY-521 模块的上拉电阻不会影响启动，并接受 `GPIO8` 板载 LED 与 SDA 共用带来的干扰风险。平衡控制对 MPU6050 稳定性要求高，首版不值得冒这个险。

PWM 能力确认：

- ESP32-C3 的 LEDC 外设有 6 个 PWM 通道，足够本项目使用。
- ESP32-C3 通过 GPIO Matrix 路由外设信号，LEDC PWM 可以输出到可用的输出 GPIO。
- `GPIO5`、`GPIO6`、`GPIO7` 都是可输出 GPIO，不是 SPI Flash、USB-JTAG 或板载 LED 专用脚。
- 所以 `GPIO6 -> AIN1`、`GPIO7 -> AIN2` 这两个脚都可以输出 PWM。为了双向推力，建议两个输入脚都按 PWM 脚使用。
- `GPIO5` 接 `nSLEEP` 只需要普通高低电平，不需要 PWM。
- `GPIO10` 暂时不接，保留为备用 GPIO/PWM。

## 8. 调试路线

1. **只测 IMU**：静止 10 秒，确认角度零偏和陀螺零偏。
2. **只测电机**：不装桨，确认 PWM、正反转、死区。
3. **测推力**：装桨后固定在电子秤上，记录 PWM 与推力，特别是正转/反转差异。
4. **手持闭环**：手拿梯子，让 PID 输出随倾角变化，但不要放手。
5. **低功率站立**：限制最大 PWM 到 30-40%，防止摔倒时打满。
6. **逐步放开限幅**：调 Kp/Kd，再考虑 Ki。
7. **固定电池位置**：调稳后不要随便移动电池，因为质心变化会改变参数。

## 9. 主要风险与规避

| 风险 | 表现 | 规避 |
|---|---|---|
| 单电机反向推力不足 | 一边能拉回，另一边拉不回 | 换对称桨/大桨/8520，或改双电机 |
| 电源噪声进 MPU6050 | 角度跳、I2C 掉线 | 电机与 MPU6050 分区，电源滤波，线短，地回路分开 |
| PCB 太软 | 高频抖动，PID 怎么调都怪 | 加厚 PCB、加宽梁、加加强条 |
| 尖脚打滑 | 梯子横向滑走，不像倒立摆 | 换金属尖脚，换接触面，增大摩擦 |
| 重心太高/太重 | 电机满功率仍救不回 | 电池下移，减重，增大螺旋桨高度 |
| ESP32-C3 启动失败 | 上电进不了程序 | 避开启动脚，检查外设上下拉 |
| 裸桨危险 | 调参时突然满功率 | 加防护圈和急停逻辑 |

## 10. 第一版硬件建议

如果现在就要下单和画第一版 PCB，我建议这样定：

- **主控**：ESP32-C3 SuperMini，插在 H 型 PCB 中部横梁上。
- **姿态传感器**：GY-521 MPU6050 模块，PCB 预留 `VCC/GND/SDA/SCL/INT` 插座。
- **电机**：8520 空心杯 + 65/75 mm 多种桨叶；如果后续实测总重很低，再退回 820。
- **驱动**：DRV8833，预留 A/B 通道并联输出焊盘。
- **供电**：1S 650 mAh 25C LiPo；原型用 5 V 升压给 SuperMini，电机 VM 可先接 5 V 并软件限幅。
- **PCB**：2.0 mm FR-4 H 型；中部横梁布电子元件；底部两个 M2 尖头螺丝作尖脚；电池用绑带槽固定，可上下调节。
- **安全**：防护圈、总电源开关、电池 ADC、摔倒停机逻辑都放进首版。

这个方案的成功率比“直接裸芯片 + 全集成电源 + 820 电机 + 只靠 PCB 尖角”高很多，也更适合课程设计迭代。
